4 
5 This is a MH-Z19B CO2 sensor library for Arduino. It has been built from scratch to support hardware
6 and software serial with a small footprint.
7 
8 The MH-Z19B is a NDIR (Non-Dispersive Infrared) type gas sensor with built-in temperature
9 compensation to measure CO2 concentration in air.

11 ![MHZ19B](https://raw.githubusercontent.com/Erriez/ErriezMHZ19B/master/extras/MHZ19B.png)

14 ## Library features
15 
16 * Small code/memory footprint
17 * Hardware and software serial interface at 9600 baud 8N1
18 * Read CO2 concentration 400..5000 ppm +/-50ppm+3% minimum 5 seconds interval
19 * Chip detection
20 * Smart warming-up detection
21 * Read firmware version
22 * Set/get range 2000 or 5000 ppm
23 * Set/get auto calibration (Automatic Baseline Correction 24h interval)
24 * Manual 400ppm calibration command
25 * CRC checks on communication protocol and timeout handling
26 * Interface for sending undocumented commands

53 ## CO2 Concentrations
54 
55 The table below displays the human impact of CO2:
56 
57 | CO2 ppm | Description |
58 | ---------- | ------------------------------------------------------------ |
59 | 0..399 | Incorrect values. Minimum value starts at 400ppm outdoor fresh air. |
60 | 400..1000 | Concentrations typical of occupied indoor spaces with good air exchange. |
61 | 1000..2000 | Complaints of drowsiness and poor air quality. Ventilation is required. |
62 | 2000..5000 | Headaches, sleepiness and stagnant, stale, stuffy air. Poor concentration, loss of attention, increased heart rate and slight nausea may also be present. |
63 | >5000 | Higher values are extremely dangerous and cannot be measured by this sensor. |

66 ## Usage
67 
68 * Operating voltage is between 4.5 and 5VDC, 150mA peak current (average < 60mA).
69 * UART pins are compatible with processors running at 3.3V without level converters.
70 * Keep sensor outside direct sunlight.
71 
72 ### Calibration
73 
74 The sensor requires an internal calibration regularly. Without it, the minimum value drifts away
75 which is noticeable after a few weeks of operation. With my experiments, the minimum value was
76 drifted to 800ppm after 3 months continues operation without a calibration.
77 
78 There are two calibration options:
79 
80 1. Automatic calibration, performed every 24 hours (default).
81 2. Manual calibration.
82 
83 ### 1. Automatic Calibration
84 
85 Automatic calibration is recommended when the sensor cannot be moved outdoor with fresh air. This
86 calibration method requires a regularly ventilated room at 400ppm, at least once in 1..3 weeks.
87 Additionally, it requires continues power-up without interruptions, otherwise the calibration data
88 will not be updated correctly.
89 
90 Automatic calibration configuration:
91 
92 * Set auto calibration on: `setAutoCalibration(true)` (Default from manufacture).
93 * Set auto calibration off: `setAutoCalibration(false)`.
94 
95 The status can be read with function `getAutoCalibration()`.
96 
97 **Note:**
98 For simplicity, this library uses the terminology `Automatic Calibration` which is identical to the
99 `ABC (Automatic Baseline Correction) logic on/off` mentioned in the datasheet.
100 
101 ### 2. Manual Calibration (400ppm)
102 
103 Procedure for manual calibration at 400ppm:
104 
105 * Turn automatic calibration off.
106 * Power the sensor up outdoor in fresh air for at least 20 minutes. (Not in a forest or a farm
107  which produces background CO2)
108 * Call `manual400ppmCalibration()` once. This will send command `0x87 Zero Point Calibration`, but
109  is not a zero calibration as stated in the datasheet. There is no nitrogen needed as this
110  calibration is performed at 400ppm.
111 
112 Now the sensor is calibrated. Repeat the sequence more often for higher accuracy.
113 
114 ### 3. Manual Calibration (SPAN)
115 
116 The datasheet also mentions a command `0x88 Span Point Calibration`. The calibration procedure is
117 not clear and therefore not implemented in this library.

120 ## MH-Z19B API
121 
122 **Initialization Software Serial**
123 
124 Use a Software Serial when no hardware serial is available. Sometimes a 3rd party library is
125 required, for example for ESP32 targets by installing `ESPSoftwareSerial`. It must be installed into
126 `.arduino15/packages/esp32/hardware/esp32/<version>/libraries/EspSoftwareSerial`, because the
127 library contains a naming conflict with existing `SoftwareSerial.h` built-in libraries.
128 
129 ```c++
130 #include <ErriezMHZ19B.h>
131 #include <SoftwareSerial.h>
132 
133 // Pin defines
134 #define MHZ19_TX_PIN 4
135 #define MHZ19_RX_PIN 5
136 
137 // Create software serial object
138 SoftwareSerial mhzSerial(MHZ19_TX_PIN, MHZ19_RX_PIN);
139 
140 // Create MHZ19B object with software serial
141 ErriezMHZ19B mhz19b(&mhzSerial);
142 ```
143 
144 **Initialization Hardware Serial**
145 
146 Any hardware serial like `Serial`, `Serial1`, `Serial2` etc can be used when supported by the CPU.
147 Multiple hardware serial ports are only available on targets like ATMEGA2560, Leonardo and SAM DUE
148 boards:
149 
150 ```c++
151 #include <ErriezMHZ19B.h>
152 
153 // Create MHZ19B object with hardware serial
154 ErriezMHZ19B mhz19b(&Serial1);
155 ```
156 
157 **General initialization**
158 
159 The optional items of the initialization sequence can be omitted.
160 
161 ```c++
162 void setup()
163 {
164  // Initialize serial
165  Serial.begin(115200);
166  Serial.println(F("\nErriez MH-Z19B CO2 Sensor example"));
167 
168  // Initialize software serial at fixed baudrate
169  mhzSerial.begin(9600);
170 
171  // Optional: Detect MH-Z19B sensor (check wiring / power)
172  while ( !mhz19b.detect() ) {
173  Serial.println(F("Detecting MH-Z19B sensor..."));
174  delay(2000);
175  };
176 
177  // Sensor requires 3 minutes warming-up after power-on
178  while (mhz19b.isWarmingUp()) {
179  Serial.println(F("Warming up..."));
180  delay(2000);
181  };
182 }
183 ```
184 
185 **Read CO2 loop**
186 
187 Read CO2 with minimum interval asynchronous function `isReady()`.
188 A good practice is to check error returns `< 0`.
189 
190 ```c++
191 void loop()
192 {
193  int16_t result;
194 
195  // Minimum interval between CO2 reads
196  if (mhz19b.isReady()) {
197  // Read CO2 from sensor
198  result = mhz19b.readCO2();
199 
200  // Print result
201  if (result < 0) {
202  // Print error code
203  switch (result) {
204  case MHZ19_RESULT_ERR_CRC:
205  Serial.println(F("CRC error"));
206  break;
207  case MHZ19_RESULT_ERR_TIMEOUT:
208  Serial.println(F("RX timeout"));
209  break;
210  default:
211  Serial.print(F("Error: "));
212  Serial.println(result);
213  break;
214  }
215  } else {
216  // Print CO2 concentration in ppm
217  Serial.print(result);
218  Serial.println(F(" ppm"));
219  }
220  }
221 }
222 ```
223 
224 **Print internal settings**
225 
226 All tests are performed with sensor version string `"0443"`.
227 
228 ```c++
229 char firmwareVersion[5];
230 
231 // Optional: Print firmware version
232 Serial.print(F(" Firmware: "));
233 mhz19b.getVersion(firmwareVersion, sizeof(firmwareVersion));
234 Serial.println(firmwareVersion);
235 
236 // Optional: Print operating range
237 Serial.print(F(" Range: "));
238 Serial.print(mhz19b.getRange());
239 Serial.println(F("ppm"));
240 
241 // Optional: Print Automatic Baseline Calibration status
242 Serial.print(F(" Auto calibrate: "));
243 Serial.println(mhz19b.getAutoCalibration() ? F("On") : F("Off"));
244 ```

256 ### Documented commands
257 
258 The following commands are documented, used and tested by the library:
259 
260 | Command | Description |
261 | ------- | -------------------------------------------- |
262 | 0x79 | Set auto calibration on/off |
263 | 0x86 | Read CO2 concentration |
264 | 0x87 | Calibration zero point at 400ppm (not 0 ppm) |
265 | 0x88 | Calibrate span point (NOT IMPLEMENTED) |
266 | 0x99 | Set detection range |
267 
268 
269 ### Not documented commands (tested)
270 
271 The following commands are **not documented**, are used and tested by the library:
272 
273 | Command | Description |
274 | ------- | -------------------------------------------- |
275 | 0x7D | Get auto calibration status (NOT DOCUMENTED) |
276 | 0x9B | Get range detection (NOT DOCUMENTED) |
277 | 0xA0 | Get firmware version (NOT DOCUMENTED) |
278 
279 More information about undocumented commands: https://revspace.nl/MH-Z19B.
280 
281 **NOTE:** Sending untested commands may damage the sensor permanently! Use at your own risk.
282 
283 ```c++
284 uint8_t response[MHZ19B_RESPONSE_LENGTH];
285 int16_t result;
286 
287 mhz19b.sendCommand(MHZ19B_CMD_NOT_DOCUMENTED, 0x00, 0x00, 0x00, 0x00, 0x00);
288 result = mhz19b.receiveResponse(response, sizeof(response));
289 ```

292 ## Library configuration
293 
294 Unfortunately, the sensor has no possibility to read warming-up status, so the library must wait
295 at least 3 minutes after reset or power-on. To speedup the boot process, macro
296 `MHZ19B_SMART_WARMING_UP` can be enabled in `ErriezMHZ19B.h` to enable smart warming-up when the
297 MCU is reset and MH-Z19B powered > 3 minutes.
